# 🚀 Sentiment Analytics Engine

A web-based **Sentiment Analysis Platform** that allows users to upload CSV datasets and analyze textual sentiment automatically using Natural Language Processing (NLP).

This system processes textual data and classifies it into **Positive, Negative, or Neutral sentiment**, helping users quickly understand opinions or emotions from datasets.

🌐 **Live Application**  
https://sentiment-analytics-engine.up.railway.app

---

# 📌 Project Overview

The goal of this project is to build a **simple, accessible web platform** where users can perform sentiment analysis without writing code.

Users can upload any **CSV dataset containing textual data**, and the system will automatically process it using NLP techniques to determine the sentiment of the text.

This project demonstrates the practical implementation of:

- Natural Language Processing
- Web Development using Flask
- Dataset handling
- Cloud deployment

---

# 🏗️ System Workflow

```
CSV Dataset
   ↓
Web Interface (User Upload)
   ↓
Flask Backend Processing
   ↓
Text Preprocessing
   ↓
Sentiment Analysis (NLTK + TextBlob)
   ↓
Sentiment Classification
   ↓
Results Output
```

# ⚙️ How to Use the Project

1️⃣ Download any **CSV dataset** containing text data from sources such as Kaggle.

2️⃣ Open the web application.

3️⃣ Upload the CSV file using the upload option.

4️⃣ The system will process the text data using NLP techniques.

5️⃣ View the **sentiment results** generated by the model.

The system automatically detects sentiment and categorizes it as:

- Positive
- Negative
- Neutral

# ☁️ Deployment

The project is hosted using **Railway**, a cloud platform that allows quick deployment of web applications.

Deployment process included:

- Preparing project dependencies using `requirements.txt`
- Connecting the GitHub repository with Railway
- Running the Flask application on Railway servers
